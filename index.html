Modul 01 · Bits & Bytes

Bits & Bytes

Alles was ein Computer speichert, verarbeitet oder überträgt, besteht auf dem untersten Level aus zwei Zuständen: Spannung an oder aus. Das sind 1 und 0. Darauf baut alles auf.

Das Bit

Ein Bit (Binary Digit = binäre Stelle) ist die kleinste mögliche Informationseinheit. Es kann genau zwei Zustände annehmen: 1 oder 0. An oder aus. Wahr oder falsch.

Ein einzelnes Bit ist für sich wenig nützlich. Erst durch Kombination mehrerer Bits entstehen aussagekräftige Werte.

Bit

1 oder 0

Kleinste Einheit. Kommt von "binary digit". Ein Bit kann nur zwei verschiedene Werte haben.

Nibble

4 Bits

Halb so groß wie ein Byte. Kann Werte von 0 bis 15 darstellen. Entspricht genau einer Hex-Stelle.

Byte

8 Bits

Die grundlegende Speichereinheit. Kann 28 = 256 verschiedene Werte annehmen (0 bis 255).

Kilobyte (KB)

1.024 Byte

Im binären System: 210 = 1.024 Byte. Im Dezimalsystem (Festplattenhersteller): 1.000 Byte.

Warum 8 Bits pro Byte?

Das ist historisch gewachsen. Frühe Computer nutzten 6 oder 7 Bits. IBM setzte 1964 mit dem System/360 den Standard auf 8 Bits, weil das reicht um alle druckbaren Zeichen plus Steuerzeichen darzustellen und gut durch 2 teilbar ist.

Was kann ein Byte darstellen?

Mit 8 Bits sind genau 256 Kombinationen möglich (2 hoch 8). Das reicht für:

Beispiel01000101 = 0x65 = 'e' = Dezimal 101 0x bedeutet: diese Zahl ist hexadezimal (Basis 16) angegeben. 0x65 ist nicht "null x 65", sondern das Präfix für eine Hex-Zahl. 6 * 16 + 5 = 101 -- das ist derselbe Wert wie Dezimal 101.
0x erklärt

0x ist ein Präfix das anzeigt: "die folgende Zahl ist hexadezimal". 0x65 bedeutet also: Hex-Zahl 65. Das entspricht Dezimal 101. Genauso zeigt 0b eine Binärzahl an: 0b01000101 = dieselbe 101.

Binär vs. Dezimal – der Unterschied

Festplattenhersteller werben mit Dezimal-GB (1 GB = 1.000.000.000 Byte). Windows zeigt aber binäre GiB (Gibibyte: 1 GiB = 1.073.741.824 Byte). Eine "1 TB"-Festplatte zeigt Windows deshalb als ~931 GB an. Das ist kein Fehler, nur zwei verschiedene Definitionen.

Modul 01 · Bits & Bytes

Zahlensysteme

Computer rechnen binär, Menschen rechnen dezimal, und Entwickler lesen Hex. Alle drei Systeme beschreiben dieselben Werte – nur mit unterschiedlichen Symbolen.

Drei Systeme, ein Wert

Dezimal (Basis 10)

0–9

Das System das Menschen seit Jahrtausenden nutzen. 10 Symbole, weil wir 10 Finger haben. Stellenwert: 1, 10, 100, 1000...

Binär (Basis 2)

0 und 1

Das System der Computer. Nur 2 Symbole, weil Hardware nur zwei Zustände kennt. Stellenwert: 1, 2, 4, 8, 16...

Hexadezimal (Basis 16)

0–9 und A–F

16 Symbole. A=10, B=11, ... F=15. Sehr kompakt: 2 Hex-Stellen = 1 Byte. Deshalb überall in der IT.

Beispiel: Wert 255Dezimal: 255 Binär: 11111111 -- 8 Bits, alle auf 1 Hexadezimal: 0xFF -- F=15, also 15*16 + 15 = 255 Dezimal: 65 Binär: 01000001 -- 64+1 = 65 Hexadezimal: 0x41 -- 4*16 + 1 = 65 = 'A' in ASCII

Wie liest man Binär?

Jede Stelle hat einen Stellenwert der sich verdoppelt: von rechts nach links 1, 2, 4, 8, 16, 32, 64, 128. Du addierst die Werte der Stellen die eine 1 haben.

Beispiel: 0b10110010Stelle: 128 64 32 16 8 4 2 1 Bit: 1 0 1 1 0 0 1 0 Wert: 128 + 0 + 32 + 16 + 0 + 0 + 2 + 0 = 178

Wie liest man Hexadezimal?

Jede Hex-Stelle entspricht genau 4 Bits (einem Nibble). Das macht Hex so praktisch: ein Byte = 2 Hex-Stellen, immer.

Hex lesen0xB2 = B * 16 + 2 = 11 * 16 + 2 = 176 + 2 = 178 -- Schnelle Umrechnung: jede Hex-Stelle = 4 Bits B = 11 = 1011 2 = 2 = 0010 B2 = 10110010 = 0b10110010 = 178

Modul 02 · Was ist eine Datei?

Dateien & Metadaten

Eine Datei ist eine benannte, geordnete Folge von Bytes, dauerhaft auf einem Speichermedium abgelegt. Das klingt simpel – aber dahinter steckt ein ganzes System.

Was ist eine Datei?

Auf dem untersten Level ist eine Datei nichts anderes als eine Sequenz von Bytes auf einem Speichermedium. Das Betriebssystem (Operating System = OS) weiß nicht was in der Datei steckt. Es verwaltet nur wo sie liegt und wie sie heißt. Den Inhalt interpretiert immer das öffnende Programm.

Wichtig

Die Dateiendung (z.B. .jpg) ist nur ein Hinweis für das Betriebssystem welches Programm die Datei öffnen soll. Sie kann umbenannt werden ohne den Inhalt zu verändern. Der echte Dateityp steckt in den ersten Bytes der Datei selbst (dazu mehr in Modul 4).

Metadaten

Metadaten (Meta = über, Daten = Informationen) sind Daten über eine Datei, nicht der Inhalt selbst. Das Dateisystem speichert sie getrennt vom eigentlichen Dateiinhalt.

Name

Dateiname + Endung

Der menschenlesbare Name. Wird vom Dateisystem gespeichert, nicht in der Datei selbst.

Größe

Dateigröße in Bytes

Wie viele Bytes der Inhalt hat. Unterschied zur "Größe auf Datenträger" (immer ein Vielfaches der Clustergröße).

Zeitstempel

Erstellt / Geändert / Geöffnet

Drei verschiedene Zeitstempel: Erstellung, letzte Änderung, letzter Zugriff. Forensisch relevant.

Berechtigungen

Wer darf was

Lesen, Schreiben, Ausführen. Unter Linux rwx, unter Windows ACL (Access Control List). Mehr dazu in Modul 6.

Speicherort

Inode / MFT-Eintrag

Zeiger auf die tatsächlichen Datenblöcke auf dem Datenträger. Der "Inhaltsverzeichnis-Eintrag" der Datei.

Attribute

Versteckt, System, Archiv

Zusätzliche Flags. Z.B. "versteckt" unter Windows, "system" für Betriebssystemdateien.

Inode – der interne Datei-Eintrag (Linux)

Unter Linux verwaltet das Dateisystem jede Datei über einen Inode (Index Node = Index-Knoten). Der Inode enthält alle Metadaten außer dem Dateinamen. Der Dateiname ist nur ein Verweis auf den Inode.

Linux Terminal# Inode-Nummer einer Datei anzeigen ls -i datei.txt # Ausgabe: 1234567 datei.txt # Alle Metadaten einer Datei anzeigen stat datei.txt # Zeigt: Größe, Blöcke, Zugriffsrechte, Inode, Zeitstempel
Warum ist das relevant?

Wenn du eine Datei "löschst", wird meist nur der Inode-Eintrag als frei markiert. Die eigentlichen Daten bleiben auf dem Datenträger bis sie überschrieben werden. Das ist der Grund warum gelöschte Dateien mit Forensik-Tools wiederhergestellt werden können.

MFT – Master File Table (Windows/NTFS)

Windows NTFS (New Technology File System) nutzt statt Inodes die MFT (Master File Table = Haupt-Dateitabelle). Jede Datei hat einen MFT-Eintrag mit allen Metadaten. Bei kleinen Dateien (< ~700 Byte) wird der Inhalt direkt im MFT-Eintrag gespeichert.

Größe vs. Größe auf Datenträger

Windows zeigt zwei Größen: "Größe" ist die tatsächliche Dateigröße in Bytes. "Größe auf Datenträger" ist immer ein Vielfaches der Clustergröße (z.B. 4 KB). Eine 1-Byte-Datei belegt trotzdem 4 KB auf der Festplatte. Der Rest ist verschwendeter Platz (Slack Space = Leerraum).

Modul 02 · Was ist eine Datei?

Dateisysteme

Ein Dateisystem ist die Struktur die ein Betriebssystem nutzt um Daten auf einem Speichermedium zu organisieren. Ohne Dateisystem wäre eine Festplatte nur ein riesiger Haufen Bytes ohne Anfang und Ende.

Was macht ein Dateisystem?

Ein Dateisystem löst vier Grundprobleme:

Organisation

Ordner & Dateien

Hierarchische Struktur aus Verzeichnissen und Dateien. Wo fängt was an, wo hört es auf.

Adressierung

Wo liegt was

Welche Blöcke auf dem Datenträger gehören zu welcher Datei. Zuordnung von Namen zu Speicherorten.

Freispeicher

Was ist noch frei

Verwaltung welche Blöcke belegt und welche frei sind. Ohne das könnte man nichts schreiben.

Metadaten

Infos über Dateien

Name, Größe, Zeitstempel, Berechtigungen – alles was nicht der Dateiinhalt selbst ist.

Die wichtigsten Dateisysteme

NTFS – New Technology File System

Das Standard-Dateisystem von Windows seit Windows NT (1993). Heute auf praktisch allen Windows-Systemen.

Stärken
  • Dateigrößen bis 16 EB (Exabyte) theoretisch
  • Journaling (Fehlertoleranz bei Stromausfall)
  • Zugriffsrechte (ACL = Access Control List)
  • Verschlüsselung (EFS = Encrypting File System)
  • Komprimierung einzelner Dateien
Schwächen
  • Nur von Linux mit zusätzlichem Treiber beschreibbar
  • macOS kann nur lesen, nicht schreiben
  • Für USB-Sticks überdimensioniert

FAT32 – File Allocation Table 32

Ein altes aber universelles Dateisystem (1996). Wird heute vor allem für USB-Sticks und SD-Karten genutzt wegen maximaler Kompatibilität.

Kritische Einschränkung

FAT32 unterstützt keine einzelne Datei größer als 4 GB (genau: 4 GB minus 1 Byte). Das ist ein hartes Limit im Dateisystemdesign. Ein 8 GB großes Video passt deshalb nicht auf einen FAT32-formatierten USB-Stick.

Stärken
  • Von Windows, Linux, macOS, Kameras, Radios lesbar
  • Einfach und robust
  • Geringe Overhead-Größe
Schwächen
  • Max. 4 GB pro Datei
  • Max. 32 GB Partitionsgröße (Windows-Formatierung)
  • Kein Journaling, keine Berechtigungen

exFAT – Extended File Allocation Table

Microsofts Nachfolger für FAT32 (2006). Keine 4-GB-Grenze mehr, trotzdem breit kompatibel. Heute Standard für große SD-Karten (SDXC).

ext4 – Fourth Extended Filesystem

Das Standard-Dateisystem von Linux seit 2008. Robust, schnell, mit Journaling.

Stärken
  • Sehr zuverlässig und erprobt
  • Journaling, große Dateien, große Partitionen
  • Standardmäßig auf fast allen Linux-Systemen
Schwächen
  • Windows kann ext4 nicht lesen (ohne Drittanbieter-Software)
  • macOS auch nicht nativ

Was ist Journaling?

Journaling (Journal = Tagebuch) bedeutet: bevor das Dateisystem eine Änderung vornimmt, schreibt es sie zuerst in ein Journal. Wenn der Strom während einer Schreiboperation ausfällt, kann das Dateisystem beim nächsten Start das Journal durchgehen und unvollständige Operationen rückgängig machen. Ohne Journaling können Dateien nach einem Absturz korrumpiert sein.
